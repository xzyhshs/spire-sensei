## 附录：Brainstorming Skill 使用说明

> 这是你第一次使用 Superpowers 的 brainstorming skill。以下是每一步的说明，帮助你理解整个流程。

### Brainstorming 的 9 个步骤

| 步骤 | 做什么 | 怎么做 | 谁主导 |
|------|--------|--------|--------|
| **1. 探索项目上下文** | AI 检查当前项目目录、文件、git 状态 | AI 自动执行，用 ls/glob/grep 查看项目 | AI |
| **2. 提供视觉伴侣** | AI 询问是否需要浏览器端的可视化辅助（流程图/UI 草图） | AI 单独发一条消息询问，不夹带其他内容 | AI 问，你决定 |
| **3. 逐步澄清需求** | AI 一次只问一个问题，逐个确认你的意图、约束、成功标准 | AI 提问 → 你回答 → AI 再问下一个。优先用选择题 | AI 主导提问 |
| **4. 提出 2-3 种方案** | AI 根据你的需求，列出不同实现方案及各自优劣 | AI 列举方案 + 推荐 + 理由 | AI |
| **5. 分块展示设计** | AI 把完整设计拆成小块，每次展示一块等你确认 | AI 展示一块 → 你审阅 → 确认/提修改 → AI 展示下一块 | 你逐块审阅 |
| **6. 写设计文档** | 把确认的设计写成正式 Markdown 文档，存入项目 | AI 写入 `docs/superpowers/specs/YYYY-MM-DD-<主题>-design.md` | AI |
| **7. 设计自查** | AI 自检设计文档：有无占位符、矛盾、模糊表述、范围膨胀 | AI 自己扫描文档，发现问题自己修复 | AI |
| **8. 你审阅文档** | AI 告诉你文档路径，等你看完确认/提出修改 | AI 提示 → 你打开文件阅读 → 回复确认或修改意见 | **你主导** |
| **9. 进入实现计划** | AI 调用 writing-plans skill，把设计拆成可执行的 bite-sized 任务 | AI 自动调用下一个 skill | AI |

### 关键原则

- **一次只问一个问题** — 不糊一堆问题让你填
- **设计未获批准，绝不写代码** — 这是 brainstorming 的硬性门禁
- **你随时可以回退修改** — 如果某块设计你觉得不对，直接说
- **brainstorming 之后只有一条路** — writing-plans，不能跳到其他实现 skill

## 一、项目定位

**Spire Sensei** 是一个 Windows 桌面客户端，为杀戮尖塔新手玩家提供 AI 导师指导。玩家通过截图和文字与 AI 交互，客户端将每局游戏状态以 MD 快照文件保存在本地，确保 AI 始终掌握完整上下文。

**关键约束：**
- 客户端不提供 AI 服务，用户需自行填入 API Key
- 推荐使用 DeepSeek API，支持其他 OpenAI 兼容接口
- 项目在 GitHub 开源

---

## 二、技术选型

| 层级 | 选择 | 原因 |
|------|------|------|
| 桌面框架 | **Electron** | Web 技术做 UI，社区成熟，`frontend-design` skill 可复用 |
| 前端 | **React** | Electron 标配，生态丰富 |
| 后端 | Electron Main Process (Node.js) | 文件读写、API 调用 |
| 数据存储 | **本地 Markdown 文件** | 人类可读，Git 友好，用户可手动编辑 |
| AI API | **DeepSeek Vision + Chat** | 支持图片识别 + 文字对话，价格低 |

### 布局要点

- **左侧面板**（约 35% 宽度）：可视化仪表盘，从 MD 快照实时渲染
- **右侧面板**（约 65% 宽度）：始终可见的聊天框

- **底部控制栏**：教学深度切换 + 教练风格切换

## 四、聊天输入设计

### 消息发送（无模式）

| 操作 | 方式 | 效果 |
|------|------|------|
| 单独发文字 | 输入文字 → 回车 | AI 据当前快照 + 文字回复 |
| 单独发截图 | Ctrl+V 粘贴 / 拖入 → 回车 | AI 据截图 + 当前快照回复 |
| 图文混发 | 粘贴截图 + 输入文字 → 回车 | AI 据截图 + 文字 + 快照综合回复 |

### AI 不确定性处理

当 AI 无法从消息判断用户意图时，主动反问。例如：
- 用户只发截图 → AI 自问："是问选牌？问出牌顺序？还是更新卡组？"→ 问用户确认
- 用户说"这个怎么样"→ AI 结合截图理解上下文，不确定就问

### 技术细节
- 截图在发送前自动压缩（降低 API 费用）
- 图片以 base64 内联或本地临时路径发送给 Vision API

## 五、MD 快照文件格式

每局游戏对应一个 MD 文件，存储在 `games/` 目录下。文件名格式：`{角色}-{创建时间}.md`，如 `铁甲战士-20260513-203000.md`。

```markdown
---
character: 铁甲战士
floor: 12
hp: 45/72
gold: 188
act: 2
created: 2026-05-13T20:30:00+08:00
updated: 2026-05-13T21:15:00+08:00
---

# 卡组 (15)
- [ ] 打击 x4
- [ ] 防御 x4
- [x] 痛击+ x1
- [ ] 旋风斩 x1
- [ ] 耸肩无视 x1
- [x] 祭品+ x1
- [ ] 武装 x1
- [ ] 飞剑回旋镖 x1
- [ ] 死亡收割 x1

# 遗物 (4)
- 痛楚印记
- 皇家枕套
- 弹珠袋
- 开心小花

# 药水 (2)
- 格挡药水
- 能量药水

# 当前选项
- 抓牌: 燃烧 / 双发 / 震波
```

### 字段说明

| 区块 | 说明 |
|------|------|
| `---` YAML front matter | 机器解析快，存角色/层数/血量/金币/时间戳 |
| `character` | 角色名：铁甲战士 / 静默猎手 / 故障机器人 / 观者 |
| `floor` | 当前层数 |
| `hp` | 当前血量/最大血量 |
| `gold` | 金币数 |
| `act` | 第几幕 (1/2/3/4) |
| `# 卡组` | `[ ]` 未升级 / `[x]` 已升级（敲过），`xN` 数量 |
| `# 遗物` | 纯列表 |
| `# 药水` | 纯列表 |
| `# 当前选项` | 临时区块，AI 建议后被清洗，用于告诉 AI 当前面临的选择 |

### 更新机制

AI 回复中包含特殊标记块来更新状态：

\`\`\`json state
{
  "hp": "38/72",
  "gold": 200,
  "floor": 13,
  "addCards": ["燃烧"],
  "removeCards": [],
  "addRelics": [],
  "options": "战斗: 选择燃烧"
}
\`\`\`

- 客户端解析此 JSON → 更新 MD 快照 → 刷新左侧面板
- 如果 AI 没有输出此块，快照不变
- 解析失败时，保留旧快照并在日志中记录错误

## 六、教练风格 & 教学深度

### 教练风格（只改说话风格，不改游戏知识）

预设角色（锁定不可编辑）：
| 角色 | 风格描述 |
|------|----------|
| 默认 | 正常 AI 助手风格 |
| 卢本伟 | 口语化，东北腔，爱说"兄弟""干就完了" |
| 东北雨姐 | 东北方言，爽朗豪迈 |
| 特朗普 | 夸张自信，喜欢用最高级形容词 |

自定义角色：
- 用户可自由编辑风格描述
- 存储在用户配置文件中

实现方式：所有风格只是一段 system prompt 片段，注入到 AI 上下文中。

### 教学深度

| 模式 | 效果 |
|------|------|
| **深度教学** | 详细解释选牌原则、出牌顺序、为什么选/不选、长期策略考量 |
| **学一点点** | 只给结论，不解释原因，一句话说完 |

实现方式：与风格一样，是一段 system prompt 片段。

## 八、截图处理流程

```
用户粘贴/拖入截图
  ↓
客户端压缩图片（降低 API 费用）
  ↓
客户端读取当前 MD 快照 → 拼入上下文
  ↓
发送消息 [截图 + 快照上下文 + 教练风格 + 教学深度]
  ↓
调用 AI API（图片 + 文字合并发送）
  ↓
AI 响应（文本 + 可选 state 更新 JSON 块）
  ↓
客户端解析 state JSON → 更新 MD 快照 → 刷新左侧面板
  ↓
显示 AI 回复在聊天区域
```

## 九、游戏管理

### 新游戏
1. 用户点击「新游戏」
2. 选择角色（战士/猎人/机器人/观者）
3. 客户端生成 `games/{角色}-{时间}.md`
4. 切换为当前活跃文件

### 切换游戏
1. 用户点击「切换游戏」
2. 显示 `games/` 下所有 MD 文件列表
3. 选择后切换为活跃文件，左侧面板刷新

### 文件存储
- 所有 MD 文件存在 `games/` 目录
- 单文件对应单局游戏
- 用户可手动打开 MD 文件编辑

## 十、完整功能清单

| 模块 | 功能 |
|------|------|
| 左侧仪表盘 | 实时可视化游戏状态（角色/血量/金币/层数/卡组/遗物/药水/当前选项） |

| 截图处理 | 压缩 → 拼入上下文 → AI 响应 → 解析 JSON 更新 MD |
| 多局管理 | 手动新游戏 → 创建文件，手动切换活跃文件 |
| 覆盖场景 | 抓牌建议、路线规划、商店决策、战斗策略、遗物选择 |
